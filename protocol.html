MQTT 协议交互流程

1. 连接建立流程

Client
Broker
CONNECT
clientId, username, password, will, cleanSession
CONNACK
returnCode: 0=成功 
# CONNECT 报文示例
Packet Type: 0001 (CONNECT)
Flags: 0000
Remaining Length: variable

Payload:
  - Protocol Name: "MQTT"
  - Protocol Level: 4 (MQTT 3.1.1) or 5 (MQTT 5.0)
  - Connect Flags:
    * Clean Session: 1
    * Will Flag: 0
    * Will QoS: 00
    * Will Retain: 0
    * Password Flag: 1
    * Username Flag: 1
  - Keep Alive: 60 秒
  - Client ID: "my-client-001"
  - Username: "user123"
  - Password: "pass456"

2. 发布/订阅流程（QoS 1）

Publisher
Broker
Subscriber
SUBSCRIBE
topic: "sensors/temp", QoS: 1
SUBACK
Message ID: 1, Return QoS: 1
PUBLISH
topic: "sensors/temp", payload: "25.5°C"
PUBACK
Message ID: 100
PUBLISH
delivered to subscriber
PUBACK
from subscriber

3. QoS 2 完整握手流程

Publisher
Broker
Subscriber
PUBLISH (QoS 2)
Message ID: 123
PUBREC
Message ID: 123
PUBREL
Message ID: 123
PUBCOMP
Message ID: 123

📌 QoS 2 四次握手说明

  1. PUBLISH: 发布者发送消息
  2. PUBREC: Broker 确认收到（Publish Received）
  3. PUBREL: 发布者释放消息（Publish Release）
  4. PUBCOMP: Broker 确认完成（Publish Complete）

4. 遗嘱消息（Last Will）

5. 心跳保活机制

Client
Broker
CONNECT
Keep Alive: 60s
CONNACK
PINGREQ
< 60s 内发送
PINGRESP

⚠️ 保活超时

如果 Broker 在 1.5 倍 Keep Alive 时间内未收到任何消息，将断开连接并触发遗嘱消息（如果已设置）。
