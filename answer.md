二、MQTT 底层传输协议

核心答案：TCP/IP 为主，WebSocket 为辅

```
┌─────────────────────────────────────────┐
│           应用层：MQTT协议               │
│    (发布订阅/遗嘱/QoS/保持会话)          │
├─────────────────────────────────────────┤
│           传输层：TCP / WebSocket       │
│    (可靠传输/端口1883/8883/8083)        │
├─────────────────────────────────────────┤
│           网络层：IP (IPv4/IPv6)        │
├─────────────────────────────────────────┤
│         数据链路层/物理层                │
│    (以太网/WiFi/4G/5G/LoRa/Zigbee)     │
└─────────────────────────────────────────┘
```

---

1. TCP（主要方式）

```
┌─────────┐                      ┌─────────┐
│  客户端  │  ═══════════════════  │  Broker │
│         │      TCP三次握手        │         │
│         │  ───────────────────>  │         │
│         │      SYN               │         │
│         │  <───────────────────  │         │
│         │      SYN-ACK           │         │
│         │  ───────────────────>  │         │
│         │      ACK               │         │
│         │                      │         │
│         │  ═══════════════════  │         │
│         │      MQTT连接建立       │         │
│         │  ───────────────────>  │         │
│         │      CONNECT           │         │
│         │  <───────────────────  │         │
│         │      CONNACK           │         │
│         │                      │         │
│         │  ═══════════════════  │         │
│         │      保持长连接         │         │
│         │      (心跳保活)         │         │
│         │  <───PINGREQ/PINGRESP──>         │
│         │                      │         │
└─────────┘                      └─────────┘

默认端口：
• 1883：MQTT over TCP（明文）
• 8883：MQTT over TLS/SSL（加密）
• 8884：MQTT over WebSocket Secure
```

为什么用TCP？
- 可靠传输：内置重传机制，保证数据不丢
- 有序到达：消息按发送顺序到达
- 长连接支持：适合物联网设备持续在线
- 流量控制：防止网络拥塞

四、关键总结

问题	答案	
Broker部署在哪？	云端（大规模）、边缘（低延迟）、混合（边云协同）	
底层用什么协议？	TCP为主，WebSocket为辅，TLS加密，UDP扩展	
默认端口？	1883(TCP)、8883(TLS)、8083(WS)、8084(WSS)	
为什么选择TCP？	可靠、有序、长连接，契合物联网持续在线需求	
WebSocket优势？	浏览器原生支持，穿透防火墙，适合Web应用	

MQTT的分层设计使其既能运行在资源受限的传感器上（通过MQTT-SN over UDP），也能支撑百万级并发的云原生应用（通过TCP/TLS），展现了极强的适应性和扩展性。
